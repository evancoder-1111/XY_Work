# 环境检查清单

## 步骤 6.1：环境准备和依赖安装

### 1. 验证 Node.js 版本（>= 18.0.0）

```bash
node --version
```

如果版本低于 18.0.0，请更新 Node.js：
- 访问 https://nodejs.org/zh-cn/ 下载 LTS 版本
- 或使用 nvm 管理多个 Node.js 版本

### 2. 验证 Java 版本（>= 17）

```bash
java -version
```

如果版本低于 17，请更新 Java：
- 访问 https://www.oracle.com/java/technologies/downloads/ 下载 Java 17 或更高版本

### 3. 验证 MySQL 服务运行状态

```bash
# Windows
net start MySQL

# 或检查服务状态
sc query MySQL
```

确保 MySQL 服务正在运行，并且可以连接。

### 4. 安装前端依赖

```bash
cd frontend/XY_Portal_Frontend
npm install
```

验证安装成功：
- 检查 `node_modules` 目录是否存在
- 检查是否有错误信息

### 5. 验证后端 Maven 依赖

```bash
cd backend/XY_Portal
mvn dependency:resolve
```

验证依赖下载成功：
- 检查是否有错误信息
- 检查本地 Maven 仓库（通常在 `~/.m2/repository`）

### 6. 检查依赖安装是否成功

**前端检查**：
- [ ] `node_modules` 目录存在且完整
- [ ] 无依赖冲突或版本问题
- [ ] 可以运行 `npm run dev` 无错误

**后端检查**：
- [ ] Maven 依赖下载成功
- [ ] 无依赖冲突
- [ ] 可以运行 `mvn compile` 无错误

## 常见问题

### npm 安装失败

如果遇到证书问题：
```bash
npm config set strict-ssl false
```

或使用国内镜像：
```bash
npm config set registry https://registry.npmmirror.com
```

### Maven 下载依赖慢

配置 Maven 使用国内镜像，编辑 `~/.m2/settings.xml`：
```xml
<mirrors>
  <mirror>
    <id>aliyunmaven</id>
    <mirrorOf>*</mirrorOf>
    <name>阿里云公共仓库</name>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```
