# XY Work 统一数字门户 - 移动端和Web端用户体验设计

## 1. 设计概述

### 1.1 文档目的

本文档专注于 XY Work 统一数字门户的移动端和 Web 端用户体验设计，基于项目现有文档进行精简和聚焦，为开发团队提供清晰的双端实现指导。

### 1.2 核心功能范围

- **统一登录**：SSO 单点登录（OAuth2/SAML）
- **工作台**：角色化工作台，自定义系统入口
- **入口管理**：管理员配置系统入口（名称、图标、描述、URL）
- **任务通知**：待办任务与通知聚合

### 1.3 设计原则

- **一致性**：双端保持核心功能一致，交互模式统一
- **适配性**：根据设备特性优化布局和交互
- **高效性**：减少操作步骤，提升使用效率
- **可访问性**：符合 WCAG 2.1 AA 级标准

## 2. 设备与场景分析

### 2.1 Web 端使用场景

**主要设备**：
- 桌面电脑（≥1200px）
- 笔记本电脑（≥1200px）
- 大屏显示器（≥1920px）

**使用场景**：
- 办公室日常工作
- 长时间使用，多任务处理
- 需要查看大量信息和数据
- 需要精确操作和批量处理

**用户行为特征**：
- 鼠标操作，精确点击
- 键盘快捷键使用
- 多窗口切换
- 长时间专注工作

### 2.2 移动端使用场景

**主要设备**：
- 智能手机（320px - 768px）
- 平板设备（768px - 1024px）

**使用场景**：
- 移动办公，随时随地访问
- 快速查看通知和待办
- 紧急任务处理
- 碎片化时间使用

**用户行为特征**：
- 触摸操作，手势交互
- 单手操作习惯
- 快速浏览，快速操作
- 网络环境不稳定

## 3. 响应式设计策略

### 3.1 断点定义

| 设备类型 | 屏幕宽度 | 布局策略 | 导航方式 |
|---------|---------|---------|---------|
| 移动设备 | < 768px | 单列布局 | 底部导航 + 侧边抽屉 |
| 平板设备 | 768px - 1199px | 双列布局 | 侧边栏可折叠 |
| 桌面设备 | ≥ 1200px | 多列布局 | 顶部导航 + 侧边栏 |

### 3.2 布局适配

**工作台布局**：
- **Web 端**：网格布局，3-4 列展示系统入口卡片
- **平板端**：2-3 列网格布局
- **移动端**：单列列表，卡片式展示

**管理页面布局**：
- **Web 端**：完整数据表格，多列信息展示
- **平板端**：精简表格，支持横向滚动
- **移动端**：列表卡片化，详情页独立展示

## 5. 交互设计差异

### 5.1 导航方式

**Web 端**：
- 顶部导航栏 + 左侧导航栏
- 面包屑导航
- 键盘快捷键（Ctrl+K 搜索）
- 鼠标悬停显示子菜单

**移动端**：
- 顶部导航栏（简化）
- 底部导航栏（核心功能）
- 侧边抽屉菜单
- 手势操作（左滑返回）

### 5.2 操作方式

**Web 端**：
- 鼠标点击、悬停、拖拽
- 键盘快捷键
- 右键菜单
- 批量选择（Ctrl/Shift）

**移动端**：
- 触摸点击、长按
- 滑动操作（左滑删除、下拉刷新）
- 手势操作（捏合缩放）
- 单指操作优化

### 5.3 反馈机制

**Web 端**：
- 悬停状态变化
- 点击动画效果
- Toast 提示（右上角）
- 模态对话框

**移动端**：
- 触摸反馈（震动）
- 点击涟漪效果
- Snackbar 提示（底部）
- 全屏对话框

## 6. 性能优化策略

### 6.1 Web 端优化

- **首屏加载** < 2秒
- **代码分割**：按路由懒加载
- **图片优化**：WebP 格式，懒加载
- **缓存策略**：静态资源长期缓存
- **预加载**：预测用户操作，预加载资源

### 6.2 移动端优化

- **首屏加载** < 3秒（考虑网络环境）
- **资源压缩**：代码和图片压缩
- **离线支持**：Service Worker 缓存
- **网络优化**：请求合并，减少请求次数
- **图片适配**：根据设备像素比提供不同尺寸

## 7. 可访问性设计

### 7.1 Web 端

- **键盘导航**：所有功能支持键盘操作
- **焦点管理**：清晰的焦点指示
- **屏幕阅读器**：语义化 HTML，ARIA 标签
- **色彩对比度**：文本与背景 ≥ 4.5:1
- **字体大小**：支持浏览器缩放

### 7.2 移动端

- **触摸目标**：最小 44×44px
- **手势替代**：提供按钮替代手势操作
- **文字大小**：支持系统字体大小设置
- **高对比度**：支持系统高对比度模式
- **语音输入**：支持语音输入

## 8. 设计规范

### 8.1 色彩规范

**主色调**：
- 主色：#409EFF（Element Plus 默认蓝）
- 成功：#67C23A
- 警告：#E6A23C
- 危险：#F56C6C
- 信息：#909399

**中性色**：
- 文字主色：#303133
- 文字常规色：#606266
- 文字次要色：#909399
- 边框色：#DCDFE6
- 背景色：#F5F7FA

### 8.2 字体规范

**Web 端**：
- 主字体：PingFang SC（Mac）、Microsoft YaHei（Windows）
- 标题字号：20px / 18px / 16px
- 正文字号：14px
- 辅助文字：12px

**移动端**：
- 主字体：系统默认字体
- 标题字号：18px / 16px
- 正文字号：15px（iOS） / 14px（Android）
- 辅助文字：12px

### 8.3 间距规范

**基础单位**：8px

**间距等级**：
- 极小：4px
- 小：8px
- 中：16px
- 大：24px
- 极大：32px

**Web 端**：卡片内边距 16px，卡片间距 16px
**移动端**：卡片内边距 12px，卡片间距 12px

## 9. 组件设计规范

### 9.1 按钮

**Web 端**：
- 高度：32px / 40px
- 最小宽度：80px
- 圆角：4px
- 状态：正常 / 悬停 / 点击 / 禁用

**移动端**：
- 高度：44px（便于触摸）
- 最小宽度：120px
- 圆角：8px
- 状态：正常 / 点击 / 禁用

### 9.2 输入框

**Web 端**：
- 高度：32px / 40px
- 内边距：12px
- 边框：1px solid #DCDFE6
- 圆角：4px

**移动端**：
- 高度：44px
- 内边距：12px
- 边框：1px solid #DCDFE6
- 圆角：8px
- 字体大小：16px（避免 iOS 自动缩放）

### 9.3 卡片

**Web 端**：
- 圆角：4px
- 阴影：0 2px 12px rgba(0,0,0,0.1)
- 悬停阴影：0 4px 16px rgba(0,0,0,0.15)

**移动端**：
- 圆角：8px
- 阴影：0 2px 8px rgba(0,0,0,0.1)
- 点击反馈：轻微缩放动画

## 11. 实施建议

### 11.1 开发优先级

**第一阶段**：Web 端核心功能
- 登录页面
- 工作台页面
- 入口管理页面（管理员）

**第二阶段**：移动端适配
- 响应式布局
- 移动端交互优化
- 触摸手势支持

**第三阶段**：体验优化
- 性能优化
- 动画效果
- 可访问性增强

### 11.2 技术实现建议

**Web 端**：
- 使用 Element Plus 组件库
- Vue 3 + TypeScript
- 响应式布局使用 CSS Grid / Flexbox
- 路由懒加载

**移动端**：
- 使用 Element Plus Mobile 或 Vant
- 触摸事件处理
- 手势库（如 @vueuse/gesture）
- PWA 支持（可选）
