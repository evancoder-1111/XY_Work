# XY Work 统一数字门户 - 图标与图片资源规范

## 1. 图标规范

### 1.1 图标格式

- **首选格式**：SVG (Scalable Vector Graphics)
  - 优势：可缩放、体积小、清晰度高
  - 使用场景：所有UI图标、Logo、装饰图形
- **备选格式**：PNG
  - 用于不支持SVG的特殊场景
  - 提供2x和3x倍图支持高清显示

### 1.2 图标尺寸规范

| 尺寸 | 用途 | 示例 |
|-----|------|------|
| 16px | 文本内图标、小按钮图标、表格操作图标 | 编辑、删除、查看图标 |
| 24px | 导航菜单图标、功能按钮图标、状态指示图标 | 首页、设置、通知图标 |
| 32px | 卡片标题图标、工具栏图标 | 搜索、筛选、排序图标 |
| 48px | 系统入口图标、欢迎页图标 | 各系统Logo、功能模块图标 |
| 64px | 大尺寸功能图标、登录页图标 | 登录、注册、SSO登录图标 |

### 1.3 图标风格

- **主风格**：线性图标 (Stroke Icon)
  - 线条粗细：1.5px
  - 线条圆角：2px
  - 填充色：主色或文字颜色
- **特殊场景**：面性图标 (Filled Icon)
  - 用于强调或选中状态
  - 保持与线性图标一致的视觉比例

### 1.4 图标命名规范

- 使用英文命名，小写字母
- 单词之间使用连字符(-)分隔
- 命名应具有明确的语义
- 示例：`user-avatar.svg`, `settings-cog.svg`, `delete-trash.svg`

### 1.5 图标库管理

- 集中管理在`src/assets/icons`目录
- 按功能模块分类存放
- 建立图标索引文档，方便查找和使用

## 2. 图片资源规范

### 2.1 图片格式

| 格式 | 用途 | 压缩率 | 适用场景 |
|-----|------|-------|----------|
| JPG | 照片、复杂图像 | 高 | 用户头像、产品图片 |
| PNG | 需要透明背景的图像 | 中 | 图标、装饰元素 |
| SVG | 矢量图形 | 高 | Logo、图标、装饰图形 |
| WebP | 现代图像格式 | 极高 | 普通图片、装饰图片 |

### 2.2 图片尺寸规范

- **用户头像**：
  - 标准尺寸：64x64px
  - 大尺寸：128x128px
  - 缩略图：32x32px
- **背景图片**：
  - 登录页：1920x1080px
  - Banner：1200x400px
  - 普通背景：1000x600px
- **产品图片**：
  - 标准尺寸：400x300px
  - 缩略图：200x150px

### 2.3 图片质量规范

- 压缩率：
  - JPG：70%-80%质量
  - PNG：使用PNG-8或PNG-24，根据需求选择
  - WebP：75%质量，提供更好的压缩比
- 单个图片最大体积限制：
  - 图标：< 10KB
  - 普通图片：< 200KB
  - Banner/背景图：< 500KB

### 2.4 图片命名规范

- 使用英文命名，小写字母
- 单词之间使用连字符(-)分隔
- 命名格式：`描述-尺寸-版本.格式`
- 示例：`login-background-1920x1080-v1.jpg`, `user-avatar-default-64x64.png`

## 3. 资源管理

### 3.1 目录结构

```
src/assets/
├── icons/           # SVG图标
│   ├── common/      # 通用图标
│   ├── navigation/  # 导航图标
│   ├── operation/   # 操作图标
│   └── system/      # 系统入口图标
├── images/          # 位图图片
│   ├── avatars/     # 用户头像
│   ├── backgrounds/ # 背景图片
│   ├── banners/     # Banner图片
│   └── products/    # 产品相关图片
└── logos/           # Logo文件
    ├── svg/         # SVG格式Logo
    └── png/         # PNG格式Logo
```

### 3.2 资源引入方式

- **SVG图标**：
  - 组件化引入：使用Vue组件方式
  - 图标库引入：使用Element Plus图标库
  - 直接引入：`import IconName from '@/assets/icons/icon-name.svg'

- **图片资源**：
  - 相对路径引入：`import Image from '@/assets/images/image-name.jpg'
  - 动态引入：`require('@/assets/images/' + fileName)

### 3.3 资源更新流程

1. 设计人员提供新资源
2. 开发人员按规范重命名和存放
3. 更新资源索引文档
4. 在代码中使用新资源
5. 发布前进行资源优化检查

## 4. 性能优化

### 4.1 图标优化

- 使用SVG Sprite合并多个图标
- 简化SVG路径，移除不必要的属性
- 使用图标字体或组件库减少HTTP请求

### 4.2 图片优化

- 使用适当的图片格式
- 按实际显示尺寸提供图片，避免大图小用
- 实现响应式图片，根据设备提供不同尺寸
- 使用CDN加速图片加载
- 实现图片懒加载

### 4.3 加载策略

- 关键图标和图片优先加载
- 非关键资源延迟加载
- 使用骨架屏或占位图提升用户体验
- 实现错误兜底，避免图片加载失败影响界面
