# XY Work 统一数字门户 - 组件库规则

## 1. 组件库概述

本项目基于Element Plus组件库进行二次封装和规范，确保整体界面的一致性和规范性。

## 2. 基础组件规范

### 2.1 Button 按钮

**使用规范：**
- 主要操作按钮使用`primary`类型
- 次要操作使用默认按钮
- 危险操作使用`danger`类型
- 按钮尺寸：默认使用`medium`，表单中使用`small`
- 图标按钮：图标与文字间距为8px

**示例：**
```vue
<el-button type="primary">主要按钮</el-button>
<el-button>默认按钮</el-button>
<el-button type="danger" size="small">删除</el-button>
<el-button icon="el-icon-plus">添加</el-button>
```

### 2.2 Input 输入框

**使用规范：**
- 统一使用`medium`尺寸
- 输入框宽度根据布局需求设置，默认为100%
- 表单中输入框与标签间距为12px
- 错误提示使用`error`状态，并显示错误信息

**示例：**
```vue
<el-input
  v-model="value"
  placeholder="请输入内容"
  :validate-event="false"
/>
<el-input
  v-model="value"
  placeholder="请输入内容"
  error
  clearable
/>
```

### 2.3 Form 表单

**使用规范：**
- 表单项间距为16px
- 标签宽度统一为100px
- 表单提交按钮组右对齐
- 必填项使用红色星号标记
- 验证失败时即时反馈错误信息

**示例：**
```vue
<el-form :model="form" :rules="rules" label-width="100px">
  <el-form-item label="用户名" prop="username">
    <el-input v-model="form.username" placeholder="请输入用户名" />
  </el-form-item>
</el-form>
```

### 2.4 Table 表格

**使用规范：**
- 表格行高使用`medium`
- 表头背景色使用浅灰色
- 表格内操作按钮使用文本按钮或图标按钮
- 支持列宽调整、排序、筛选功能
- 分页控件显示在表格下方

**示例：**
```vue
<el-table :data="tableData" style="width: 100%">
  <el-table-column prop="name" label="名称" min-width="120" />
  <el-table-column prop="description" label="描述" min-width="200" />
  <el-table-column label="操作" width="150" fixed="right">
    <template #default="scope">
      <el-button size="small" text>编辑</el-button>
      <el-button size="small" text type="danger">删除</el-button>
    </template>
  </el-table-column>
</el-table>
```

### 2.5 Card 卡片

**使用规范：**
- 卡片圆角为6px
- 卡片阴影使用标准阴影
- 卡片内边距为16px
- 卡片标题使用H5标签，字体大小16px

**示例：**
```vue
<el-card shadow="hover">
  <template #header>
    <div class="card-header">
      <span>卡片标题</span>
    </div>
  </template>
  <div>卡片内容</div>
</el-card>
```

### 2.6 Dialog 对话框

**使用规范：**
- 对话框标题字体大小16px
- 按钮组右对齐
- 确认按钮为`primary`类型，取消按钮为默认类型
- 宽度根据内容设置，默认40%

**示例：**
```vue
<el-dialog
  v-model="dialogVisible"
  title="对话框标题"
  width="40%"
>
  <span>对话框内容</span>
  <template #footer>
    <span class="dialog-footer">
      <el-button @click="dialogVisible = false">取消</el-button>
      <el-button type="primary" @click="dialogVisible = false">确定</el-button>
    </span>
  </template>
</el-dialog>
```

## 3. 布局组件规范

### 3.1 Layout 布局

**使用规范：**
- 顶部导航高度60px
- 侧边栏宽度240px
- 内容区域最小宽度900px
- 移动端自动调整布局

**示例：**
```vue
<el-container>
  <el-header>顶部导航</el-header>
  <el-container>
    <el-aside width="240px">侧边栏</el-aside>
    <el-main>主内容区域</el-main>
  </el-container>
</el-container>
```

### 3.2 Menu 导航菜单

**使用规范：**
- 菜单项高度40px
- 选中状态使用主色高亮
- 子菜单展开动画时间150ms
- 图标与文字间距8px

**示例：**
```vue
<el-menu
  :default-active="activeIndex"
  class="el-menu-vertical"
  @select="handleSelect"
>
  <el-menu-item index="1">
    <el-icon><Home /></el-icon>
    <span>首页</span>
  </el-menu-item>
  <el-sub-menu index="2">
    <template #title>
      <el-icon><Setting /></el-icon>
      <span>设置</span>
    </template>
    <el-menu-item index="2-1">选项1</el-menu-item>
  </el-sub-menu>
</el-menu>
```

## 4. 图标使用规范

### 4.1 图标库

- 使用Element Plus内置图标
- 系统入口图标使用48px尺寸
- 导航图标使用24px尺寸
- 功能按钮图标使用16px尺寸

### 4.2 图标命名规范

- 使用语义化的图标命名
- 保持图标风格一致
- 避免在同一区域混合使用不同风格的图标

## 5. 自定义组件规范

### 5.1 组件命名

- 组件名使用PascalCase格式
- 文件命名使用kebab-case格式
- 组件名应具有明确的语义

### 5.2 组件开发流程

1. 需求分析
2. 组件设计（API设计、样式设计）
3. 组件开发
4. 单元测试
5. 文档编写
6. 代码评审
7. 发布使用

### 5.3 组件文档规范

每个自定义组件需包含以下文档：
- 组件用途说明
- Props参数列表
- Events事件列表
- Slots插槽说明
- 使用示例

## 6. 样式覆盖规范

### 6.1 样式覆盖原则

- 尽量使用Element Plus提供的配置项
- 必要时使用CSS变量进行覆盖
- 避免使用!important
- 限定作用域，避免全局污染

### 6.2 CSS变量使用

```css
:root {
  --el-color-primary: #1890ff;
  --el-color-success: #52c41a;
  --el-color-warning: #faad14;
  --el-color-danger: #f5222d;
  --el-color-info: #1890ff;
}
```

## 7. 组件使用优先级

1. 优先使用项目自定义组件
2. 其次使用Element Plus组件
3. 避免重复开发已有功能的组件
4. 新功能优先考虑扩展现有组件
